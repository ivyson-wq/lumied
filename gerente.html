✅ Setup do Relógio Ponto

Tudo o que precisa estar configurado pra escola usar o relógio ponto eletrônico (REP) integrado ao Lumied — com cálculo de horas, espelho mensal e justificativas de acordo com a Portaria MTP 671/2021. Itens vermelhos bloqueiam o uso; amarelos não bloqueiam mas é bom resolver.

Como funciona o ponto eletrônico no Lumied
👨‍🏫
1. Funcionário bate o ponto
Funcionário aproxima o crachá ou usa biometria no relógio físico (REP) na portaria/secretaria.
📡
2. Lumied Bridge baixa o AFD
Todo dia 03:30 da manhã, o daemon na escola conecta no REP pela rede e baixa o arquivo AFD do dia anterior. Sem ação humana.
⚙️
3. Lumied processa
Parser AFD (Portaria 671) cruza cada batida com o funcionário pelo PIS, calcula horas trabalhadas e saldo (HE/débito).
📊
4. Você confere
Acesse o Espelho de Ponto mensal por funcionário. Aprove justificativas. Imprime tudo na hora pra fechar a folha.

❓ Perguntas frequentes

Preciso de internet pro relógio funcionar?
Não. O REP funciona offline e armazena as batidas localmente. O Lumied Bridge só precisa de internet (e acesso ao REP pela rede local) pra baixar o AFD uma vez por dia.

O que é PIS e por que cada funcionário precisa de um?
PIS é o número de cadastro do funcionário no INSS (12 dígitos). É a chave que liga a batida no REP ao funcionário no Lumied. O PIS aqui precisa ser exatamente o que está cadastrado no REP — incluindo zeros à esquerda.

O que é o "Lumied Bridge"?
É um pequeno programa que roda num mini-PC ou Raspberry Pi dentro da escola. Ele conecta o REP (que está atrás do roteador, sem IP público) com o Lumied (na nuvem). Mesmo programa que já controla o reconhecimento facial — você instala uma vez e vale pros dois.

E se a coleta automática falhar um dia?
No dia seguinte ela tenta de novo automaticamente. Você sempre pode forçar pelo botão ▶ Buscar agora, ou fazer upload manual do AFD que o REP gera. Erros aparecem no Setup em vermelho.

Posso usar sem o Bridge instalado?
Sim. O Bridge só automatiza o processo. Sem ele, você exporta o AFD do REP num pen drive (ou pela tela admin do REP) e faz upload manual em Importar AFD → Upload Manual uma vez por mês.

Funciona com qual relógio ponto?
Nativamente com Control iD (iDClass Bio Prox, iDFace) e com presets prontos pra Henry (Hexa, Prima, Vega), Topdata e Madis. Outras marcas funcionam configurando manualmente a URL do AFD.

📥 Importar AFD (Portaria 671)

Importe o arquivo do REP (Registrador Eletrônico de Ponto) de duas formas: manual (upload do .txt exportado) ou automática diária (Lumied Bridge busca pela rede Wi-Fi/LAN da escola).

Como funciona: o Lumied Bridge (daemon que roda no mini-PC da escola) acessa o REP pelo IP da LAN, faz login com usuário/senha e baixa o AFD do dia anterior automaticamente, todo dia às 03:30 BRT. Sem ação manual. Funciona nativamente com Control iD (iDClass Bio Prox, iDFace) e também com Henry, Topdata e similares por interface web. Pré-requisito: o Bridge precisa estar instalado e online — veja em Controle de Acesso → Lumied Bridge.

🌐 Lumied Bridge — Face ID via LAN

Daemon que roda em um mini-PC na escola e conecta os iDFace (atrás do roteador) ao SaaS via WebSocket persistente. Use quando os iDFace não tiverem IP público — marque via_bridge no dispositivo.

📦 Instalar o daemon

Em um mini-PC/servidor da LAN da escola (Linux ou Windows com Docker): 

git clone https://github.com/ivyson-wq/lumied.git
cd lumied/lumied-bridge
cp .env.example .env
# preencha o .env (use os valores abaixo):
# LUMIED_ESCOLA_ID=
# LUMIED_BRIDGE_TOKEN=
# IDFACE_PASSWORD=
docker compose up -d --build

Depois disso o daemon imprime no log a URL local (ex.: http://192.168.1.50:8765/event) — configure-a como callback no painel admin de cada iDFace. Ou marque auto-config abaixo para o daemon configurar sozinho.

📖 Como instalar o LPR na sua escola
Passo a passo pra começar a ler placas. Marque cada etapa conforme vai concluindo — fica salvo no seu navegador.
0/9
concluído
  • O Lumied Bridge precisa estar rodando no Pi/mini-PC. Confira em Acesso → Lumied Bridge: o "Daemon" tem que estar 🟢 Online.
  • Pi 5 (8GB) ou mini-PC equivalente é o ideal. Pi 4 (4GB) funciona, mas em escolas com fluxo alto de carros pode ficar apertado.
  • Câmera Intelbras (ou compatível RTSP) já instalada apontando pro portão de carros, com IP fixo na rede da escola.
  • Acesso SSH ao Pi/mini-PC. Se não souber o IP/hostname, descubra em Acesso → Lumied Bridge → Hardware do mini-PC.
  • Senha do admin da câmera Intelbras (a que você usa pra abrir o painel da câmera no navegador).

O recurso de LPR precisa de uma versão recente do daemon. Faça isso primeiro; sem essa atualização, nada do que vem abaixo funciona.

ssh pi@<ip-do-pi>
cd ~/lumied/lumied-bridge
git pull
npm install
npm run build
sudo systemctl restart lumied-bridge

Se o daemon roda em Docker:

cd ~/lumied/lumied-bridge
git pull
docker compose up -d --build

Depois confira em Lumied Bridge que o Daemon voltou a ficar 🟢 Online.

FFmpeg captura o vídeo da câmera e gera fotos para o reconhecedor analisar. gpiod só importa se você for ligar a cancela direto num pino do Pi (passo 6 opção B); se for usar webhook (Sonoff/Tasmota), pode ignorar.

sudo apt update
sudo apt install -y ffmpeg gpiod
ffmpeg -version | head -1

Resposta esperada: ffmpeg version 4.x.x ou maior. Se aparecer command not found, o apt não baixou — verifique conexão de internet do Pi.

É o serviço local que recebe a foto e devolve o texto da placa. Roda em Docker — tudo offline, nada vai pra cloud.

sudo apt install -y docker.io
sudo systemctl enable --now docker
sudo docker run -d --name codeproject-ai \
  --restart unless-stopped \
  -p 32168:32168 \
  -v cpai-data:/etc/codeproject/ai \
  codeproject/ai-server:rpi64

O primeiro boot demora uns 5 minutos (baixa o modelo de IA). Depois confira que tá no ar:

curl http://localhost:32168/v1/status/ping

Resposta esperada: {"success":true, ...}

Acesse o painel do CPAI em http://<ip-do-pi>:32168 no navegador. Aba Modules → procure License Plate Reader → clique Install. Quando ficar verde "Running", está pronto.

RTSP é o "endereço de vídeo ao vivo" da câmera. Padrão Intelbras (linha VIP / IPC / MCD):

rtsp://admin:SUA_SENHA@192.168.1.50:554/cam/realmonitor?channel=1&subtype=0

Substitua nas três partes:

  • admin / SUA_SENHA: usuário e senha do painel da câmera (não a senha do wifi).
  • 192.168.1.50: IP da câmera. Descubra:
    • Pelo painel do roteador (lista de DHCP — procure pelo MAC da câmera)
    • Pelo app Intelbras iSIC no celular
    • Por scan na rede: nmap -sn 192.168.1.0/24 (precisa sudo apt install nmap antes)
  • subtype=0: stream principal (alta qualidade, melhor pro OCR). Se o Pi não der conta, troque pra subtype=1.

Teste no Pi (substitua os valores):

ffmpeg -rtsp_transport tcp \
  -i 'rtsp://admin:SUA_SENHA@192.168.1.50:554/cam/realmonitor?channel=1&subtype=0' \
  -frames:v 1 -y /tmp/test.jpg
ls -lh /tmp/test.jpg

Se gerou um arquivo > 50KB, está OK. Pra ver a imagem do seu computador: scp pi@IP:/tmp/test.jpg ./

  1. Vá pra aba 📷 Câmeras aqui no painel
  2. Clique em + Nova câmera
  3. Preencha:
    • Nome: ex "Portão de carros"
    • RTSP URL: a URL que você testou no passo anterior
    • ALPR endpoint: deixe o padrão (http://localhost:32168/v1/vision/alpr)
    • Intervalo scan: 2000ms (a cada 2 segundos). Aumente se o Pi suar muito.
    • Confiança mínima: 0.85. Sobe pra 0.90 se aparecerem muitos falsos-positivos; abaixa pra 0.80 se rejeitar placas legítimas.
    • Câmera ativa: marcado.
  4. Salvar. A câmera começa a rodar em segundos.
  5. Vá pra aba 📋 Eventos: passe um carro pelo portão e veja se a leitura aparece.

Quando uma placa é autorizada, o daemon precisa abrir a cancela. Duas opções:

🌐 A) Webhook (recomendado)

Sem fios extras. Funciona com Sonoff Mini, Tasmota, ESP-Home, Shelly e similares.

Na edição da câmera:

  • Webhook URL: depende do firmware. Exemplos:
    • Sonoff Tasmota: http://192.168.1.100/cm?cmnd=Power%20ON
    • ESP-Home: http://192.168.1.100/switch/portao/turn_on
    • Shelly: http://192.168.1.100/relay/0?turn=on
  • Bearer token: só se o seu relé exigir auth.

Teste manual antes: curl -X POST <url> deve abrir a cancela.

🔌 B) GPIO direto

Mais barato, mas precisa de relé externo opto-isolado (ex: módulo SRD-05VDC-SL-C). Atenção a aterramento.

Na edição da câmera:

  • GPIO pin: número BCM. Comuns: 17, 27, 22, 23.
  • Pulso: 500ms (default). Aumenta se a cancela é lenta.

gpiod já foi instalado no passo 2.

⚠️ Se preencher os dois (webhook + GPIO), o GPIO tem prioridade.

ROI (Region of Interest) é o polígono da área onde leituras valem. Carros passando na rua, faixas do vizinho, gente caminhando — tudo isso é descartado quando cai fora do polígono.

  1. Aba 📷 Câmeras → clique no botão 🎯 ROI da câmera
  2. O snapshot ao vivo da câmera aparece
  3. Clique nos cantos da área onde os carros realmente passam pra abrir o portão. Mínimo 3 pontos, geralmente 4 (um quadrilátero).
  4. Salvar ROI

O polígono é salvo em coordenadas relativas (0-1), funciona em qualquer resolução de câmera. Se trocar de câmera depois, vale re-desenhar.

  1. Aba 🅿️ Placas → cadastre uma placa de teste (ex: do seu carro pessoal). Tipo do dono: "Outro" ou "Funcionário".
  2. Passe com o carro pelo portão (ou peça pra alguém passar)
  3. Aba 📋 Eventos deve mostrar a leitura com motivo ✅ Autorizado
  4. Se cancela conectada (passo 6): ela deve abrir.
  5. Cadastre uma placa fictícia (não passe pelo portão). Passe com um carro NÃO cadastrado: o evento deve ter motivo ⚠️ Não cadastrada.
